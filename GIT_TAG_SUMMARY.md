## 💡 标签管理最佳实践

### 版本号规范 (Semantic Versioning)
- `v1.0.0`: 主版本.次版本.修订版本
- `v1.0.1`: 修复bug
- `v1.1.0`: 新增功能
- `v2.0.0`: 重大更改

### 标签命名建议
- 使用 `v` 前缀: `v1.0.0`
- 遵循语义化版本
- 使用带注释的标签
- 包含详细的发布说明

### 标签管理命令
```bash
# 创建标签
git tag -a v1.0.0 -m "Release version 1.0.0"

# 删除本地标签
git tag -d v1.0.0

# 删除远程标签
git push origin --delete v1.0.0

# 检出特定标签
git checkout v1.0.0
```
